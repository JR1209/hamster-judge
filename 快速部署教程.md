# 🎯 3分钟部署教程（小白版）

想让你的仓鼠法官上线，让朋友们都能访问？跟着做就行！

## 📱 第一步：注册GitHub账号（如果已有，跳过）

1. 打开浏览器访问：https://github.com/signup
2. 输入邮箱，设置密码，创建用户名
3. 验证邮箱
4. 完成！

## 💻 第二步：上传代码到GitHub

### 方法A：使用命令行（推荐）

打开终端（Mac）或命令提示符（Windows），运行：

```bash
cd /Users/z/Desktop/仓鼠法官
bash deploy.sh
```

按照提示操作即可！

### 方法B：手动操作

1. **在GitHub创建新仓库**
   - 访问：https://github.com/new
   - Repository name 输入：`hamster-judge`
   - 选择 Public（公开）
   - 点击 "Create repository"

2. **上传文件**
   - 在新建的仓库页面
   - 点击 "uploading an existing file"
   - 把以下文件拖进去：
     - `hamster_judge.py`
     - `requirements.txt`
     - `.streamlit/config.toml`
     - `.gitignore`
   - 点击 "Commit changes"

## ☁️ 第三步：部署到Streamlit Cloud

1. **访问Streamlit Cloud**
   - 打开：https://share.streamlit.io/
   - 点击右上角 "Sign in"
   - 选择 "Continue with GitHub"（用GitHub账号登录）
   - 授权访问

2. **创建新应用**
   - 点击 "New app" 按钮
   - 或者直接访问：https://share.streamlit.io/deploy

3. **配置应用**
   填写以下信息：
   
   - **Repository**: 选择 `你的用户名/hamster-judge`
   - **Branch**: `main`
   - **Main file path**: `hamster_judge.py`
   - **App URL**: 自定义网址（如 `hamster-judge`）
     - 最终网址会是：`https://hamster-judge.streamlit.app/`

4. **点击 Deploy!**
   - 等待 2-3 分钟
   - 系统会自动安装依赖并启动应用
   - 可以看到实时日志

5. **完成！**
   - 看到 "Your app is live!" 就成功了
   - 复制网址，分享给朋友吧！

## 🎉 成功后你会看到

- ✅ 一个公开的网址，如：`https://hamster-judge.streamlit.app/`
- ✅ 任何人打开这个网址都能使用
- ✅ 无需安装任何东西
- ✅ 在手机、电脑、平板都能访问

## 🔄 如何更新应用？

修改代码后：

### 方法A：使用GitHub网页

1. 打开你的GitHub仓库
2. 点击要修改的文件（如 `hamster_judge.py`）
3. 点击铅笔图标编辑
4. 修改完点击 "Commit changes"
5. Streamlit会自动重新部署（1-2分钟）

### 方法B：使用命令行

```bash
cd /Users/z/Desktop/仓鼠法官
git add .
git commit -m "更新内容说明"
git push
```

## 💡 常见问题

### Q1: 部署失败显示"ModuleNotFoundError"？
**A:** `requirements.txt` 文件没有包含所有依赖，确保文件内容为：
```
streamlit
requests
```

### Q2: 应用能访问但很慢？
**A:** Streamlit Cloud免费版资源有限，正常现象。可以考虑：
- 优化代码
- 升级到付费版
- 使用其他部署平台

### Q3: 网址太长/不好看？
**A:** 可以：
- 在部署时自定义App URL
- 购买自己的域名并绑定
- 使用短链接服务（如 bit.ly）

### Q4: 我没有命令行基础怎么办？
**A:** 用方法B（手动上传），完全在网页上操作，不需要敲命令！

### Q5: 别人使用会消耗我的API额度吗？
**A:** 不会！当前配置是让用户输入自己的API Key。如果你想配置统一的Key，参考《部署指南.md》。

### Q6: 可以设置密码保护吗？
**A:** Streamlit免费版不支持。如需要可以：
- 升级Streamlit付费版
- 在代码中添加简单密码验证
- 使用其他支持认证的部署平台

### Q7: GitHub仓库必须是Public吗？
**A:** Streamlit Cloud免费版只支持Public仓库。Private仓库需要付费版。
